# TASK-010: 修复端口冲突导致容器名被占用的 Bug

## 需求背景
创建容器时，如果端口被占用，Docker `ContainerCreate` 成功但 `ContainerStart` 失败。此时容器名已被占用，用户换端口重新创建会报名字重复。

## 复现步骤
1. 创建容器 name=demo, port=18791（该端口已被占用）
2. 创建失败，报端口冲突
3. 修改端口为 18792，再次创建 name=demo
4. 报错：容器名 demo 已存在

## 验收标准
- 当 ContainerStart 失败时，自动清理已创建的容器，释放名字
- 用户换端口后可以用相同名字重新创建
